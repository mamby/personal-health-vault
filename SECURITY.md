# Security Policy

## Supported Versions

The latest main branch is the supported development version.

## Reporting a Vulnerability

Please report security issues privately. Do not open a public issue for vulnerabilities involving health data, encryption, local storage, or app lock behavior.

Include:

- A clear description of the issue
- Steps to reproduce
- Impact or risk level
- Platform and device details when relevant
- Any relevant screenshots, logs, or proof of concept

## Scope

This policy applies to repository code, dependencies, encrypted local vault behavior, platform app lock behavior, and documentation that affects user privacy expectations.

## Responsible Disclosure

Please give maintainers time to investigate and fix confirmed issues before public disclosure.
